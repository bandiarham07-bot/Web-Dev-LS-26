## Why This Topic

In Week 1 you got a quick, non-technical pass at "what is the internet." That was enough to get you building frontends. It is **not** enough to start building a backend — and this week we're building a real one, so we need to go one level deeper before touching any code.

---

## What You Will Learn

- **The Client-Server Model, Revisited** — not just "browser talks to server," but *what* it sends and *what* it gets back
- **HTTP Methods** — `GET`, `POST`, `PUT`, `DELETE`
- **How HTTPS Actually Works** — what TLS/SSL is doing
- **What a Backend Is** — the server, the database, and the response, and why none of that can live in the browser
- **Static vs Dynamic Content** — why your Week 2 React app needs *something* to talk to if it wants real, changing data

---

## Learning Path for This Topic

```
1. The client-server model (what a request/response actually contains)
        |
        v
2. HTTP methods (GET/POST/PUT/DELETE — the 4 verbs of the entire web)
        |
        v
3. HTTPS / TLS (why "secure HTTP" exists, and what it's protecting)
        |
        v
4. What is a backend? (the gap between "frontend" and "a real app")
```
