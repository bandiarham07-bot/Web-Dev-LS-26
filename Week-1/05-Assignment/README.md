# 🖥️ Week 1 Assignment — *Creatively Bad UI*

---

## Brief

You use dozens of tiny UI components every day — a volume slider, a battery indicator, a brightness dial, a file upload button, a notification badge.

Your task: **pick one of these everyday computer components and rebuild it as the worst possible version of itself.**

But here's the twist — your bad UI must be **intentionally, artistically bad**. Think of it as a parody.

The component must still technically **work** (it should do the thing it's supposed to do), but the way it looks, feels, and behaves should be hilariously, painfully wrong.

---

## 💡 Component Ideas

Pick **one** of these (or propose your own):

| Component | What it normally does |
|---|---|
| 🔊 Volume slider | Adjusts audio level 0–100% |
| 🔋 Battery indicator | Shows remaining charge |
| ☀️ Brightness control | Adjusts screen brightness |
| 🔔 Notification badge | Shows unread count |
| 📁 File upload button | Lets you select a file |
| 🌐 Wi-Fi signal bar | Shows connection strength |
| ⏱️ Loading bar | Shows progress of a task |
| 🕐 Clock / time display | Shows the current time |

You may also propose a different component — just keep it something a person genuinely interacts with on a computer.

## 📦 What to Submit

A **public GitHub repository** with the following structure:

```
your-name-bad-ui/
├── index.html          ← Your component lives here
├── style.css           ← All your styles
├── script.js           ← All interactivity
└── README.md           ← Your design doc (see below)
```

**Rules:**
- Static files only — HTML, CSS, and JavaScript. No frameworks, no libraries, no build tools.
- The component must be interactive (it should respond to user actions).
- The page should load by simply opening `index.html` in a browser — no server required.

---

## 📊 Grading Rubric

| Category | What we're looking for |
|---|---|
| **Creativity & Concept** | Is the bad UI intentional and original? Is there a clear "bit"? Does it show creative thought beyond just making things ugly? |
| **Technical Execution** | Does the component actually work? Is the HTML structured, CSS applied, JS functional?|
